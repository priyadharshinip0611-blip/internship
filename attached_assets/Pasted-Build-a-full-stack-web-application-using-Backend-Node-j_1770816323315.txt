====================================
ROLES
====================================

There are 3 roles:

1. ADMIN
   - Can login only (no public registration)
   - Can approve/reject farmers
   - Can view all users

2. FARMER
   - Can register
   - Must be approved by admin before login works
   - Has profile fields:
        name
        email
        password
        phone
        farmName
        location
        isApproved (default false)

3. RETAILER
   - Can register
   - Can login directly (no approval needed)
   - Has profile fields:
        name
        email
        password
        phone
        shopName
        location

====================================
DATABASE MODELS
====================================

Create a User model with:

- name (required)
- email (required, unique)
- password (hashed)
- role (enum: admin, farmer, retailer)
- phone (required)
- location (required)
- farmName (only for farmer)
- shopName (only for retailer)
- isApproved (default false, only for farmers)
- createdAt

====================================
AUTHENTICATION
====================================

- Use bcrypt to hash passwords
- Use JWT for authentication
- Protect routes with middleware
- Prevent farmer login if isApproved = false
- Admin account should be created manually when server starts:
    email: admin@platform.com
    password: admin123
